# MVP Progress Records

このディレクトリには、MVP 実装タスクまたは PR の完了記録を残します。

GitHub の PR 本文や issue tracker はレビューや進捗管理には有用ですが、後続の AI エージェントが常に過去 PR データへアクセスできるとは限りません。そのため、後続タスクが必要とする「ここまで終えたこと」「何が完成したか」「なぜそうしたか」「何を前提にしてよいか」をリポジトリ内にも保存します。

---

## 記録ルール

- タスクまたは PR が完了したら、必要に応じて [../templates/task-completion.md](../templates/task-completion.md) をコピーして記録を作成する。
- ファイル名は `phase-<phase>-<task-slug>.md` を基本とする。
  - 例: `phase-0-go-module-skeleton.md`
  - 例: `phase-2-sqlite-telemetry-queue.md`
- 1 タスク 1 ファイルを基本とし、複数 PR が同じ一覧を編集してコンフリクトする状況を避ける。
- Phase 全体の方針変更やタスク分割変更は、個別記録だけでなく [../IMPLEMENTATION_PLAN.md](../IMPLEMENTATION_PLAN.md) にも反映する。
- 完了記録は PR 本文の代替ではなく、後続タスクのためのリポジトリ内コンテキストとして扱う。

---

## 記録すべき内容

- 完了した振る舞い
- 完成した型、コンポーネント、設定、テスト
- 完成していないこと、後続タスクへ残したこと
- 採用した設計判断と理由
- 維持した責務境界
- 永続化境界と外部 I/O 境界
- 後続タスクが依存できる契約
- 実行した検証と未実施の検証
- 残るリスク
